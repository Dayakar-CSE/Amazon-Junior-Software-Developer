Introduction
Welcome to TODO Rails! In our busy lives, organizing tasks effectively can be challenging, and TODO Rails aims to simplify that process. This project guides you through building a personalized web-based task management application, where users can add, prioritize, and track their tasks seamlessly. By combining HTML, CSS, JavaScript, and Spring Boot, you’ll create an interactive, secure environment that lets users manage to-dos efficiently. This project overview will outline the goals, features, and technical components of the project as you get started.

The purpose of the project
The purpose of this project is to equip you with the hands-on experience of building a personalized to-do tracking system. You’ll consolidate skills learned in HTML, CSS, JavaScript, and Spring Boot to create a functional web application using the MVC architecture. This project serves as a bridge between conceptual knowledge and practical implementation, helping you deepen your understanding of full-stack development.

Specifically, you’ll create a web-based TODO tasks tracking system, which is personalized to the user logged, in order to test your skills on:

HTML and Thymeleaf to create the structure or “skeleton” of each webpage in TODO Rails. HTML is the language that defines what is on each page, like the text, buttons, and forms.

CSS (Cascading Style Sheets) to add color, spacing, and other design elements to make the application look good and easy to use. CSS is the driving force behind making the layout responsive, enabling you to make it display well on both a desktop and a mobile.

JavaScript to make the application interactive, allowing actions to happen on the page without reloading it entirely, as well as doing client-side validations for the different HTML forms we have in the application, and

Spring Boot to make your application interact with a database, utilize JPA, handle user logins, and control how pages work together.

The goal of the project
The primary goal of this project is to develop a web-based MVC project which allows a user to have a personalized TODO task list, as a web application, where the following are covered:

Security: Some parts of the web application are protected and need the user to be logged in for access, whereas others are open to anyone without needing to log in.

Logging in and registering: Users start by creating an account. This helps keep personal to-do lists secure and separate from others.

Adding tasks: Users access a dashboard displaying their tasks after logging in. They can add new ones, set a priority, choose a type (like personal or official), and set a due date.

Editing and deleting tasks: If users need to change task details, they can update them or delete tasks that are no longer needed.

You will create TODO Rails using several different technologies collaboratively to allow them to work in cohesion as a single unit. 

What to expect
Throughout this project, you’ll work hands-on with various web technologies and frameworks. You’ll:

Build responsive interfaces using HTML, CSS, and Thymeleaf for an appealing user experience.

Integrate JavaScript functionality for seamless client-side interactions and validations.

Use Spring Boot and JPA to create a reliable backend, handling data interactions and user authentication securely, and

Employ the MVC pattern to link frontend and backend components cohesively, following industry-standard development practices.

Technologies and components
Here’s a breakdown of the main technologies and their roles in the project:

HTML & Thymeleaf: Create templates for dynamic web pages. For example, Thymeleaf helps inject user-specific data into each page.

CSS: Style and layout customization through general and page-specific files to maintain a cohesive design.

JavaScript: Enables smooth page interactions and client-side validations without reloading the entire page.

Spring Boot: Manages backend operations, handling tasks like saving user data and managing page requests effectively.

Grading criteria
This is a graded lab. You will receive marks at the end of the lab when you submit your work. The project will be graded as follows:

Task 1 (Security Configuration): Public access is correctly configured for essential resources such as CSS files and registration pages, while other areas remain secured by user login requirements.

Task 2 (User Authentication and Registration): The application’s login and registration flows function correctly, including error handling for duplicate usernames or emails and secure password handling.

Task 3 (Task Management - CRUD Operations): The application supports full task management, allowing users to create, edit, delete, and view tasks. All task data should be stored and retrieved correctly from the database.

Task 4 (Responsive Frontend Design): The interface is responsive, with well-organized layouts and mobile-friendly designs using HTML, CSS, and JavaScript. Forms, buttons, and layouts adjust gracefully on various screen sizes.

Task 5 (Dashboard and Task Prioritization): The dashboard displays tasks based on priority levels (high, medium, low) and highlights tasks due today, showing accurate due dates and priority categorization.

Task 6 (Task Completion Tracking): The application correctly tracks and displays the status of completed and pending tasks, with visual indicators for easy recognition.

Task 7 (Real-time Date and Time Display): The server date and time are displayed accurately on the dashboard, updating in real-time for a complete user experience.

Task 8 (Logout and Session Management): The logout function effectively terminates the user session and restricts access to protected pages upon logging out.

The pass mark is 80%.You are expected to understand the code and follow the instructions presented as TODOs, numbered sequentially. Completing each TODO in order is crucial, as some tasks rely on previous ones for full functionality.

This project will test your understanding and implementation of key web development concepts across one or more of the following topics:

HTML, CSS, and JavaScript for frontend design and interactivity

Spring Boot and JPA for backend services and data management

User authentication and session management with Spring Security

MVC architecture for structured and efficient code organization

Responsive design principles for mobile and desktop interfaces

RESTful API design and implementation

Conclusion 
With TODO Rails, you’re building more than a to-do app—you’re creating a foundation in web development. This project combines essential technologies in a cohesive, hands-on way. By the end, you’ll have a deeper understanding of MVC design, frontend-backend interactions, and the synergy of web development tools to create practical, user-centric applications.
