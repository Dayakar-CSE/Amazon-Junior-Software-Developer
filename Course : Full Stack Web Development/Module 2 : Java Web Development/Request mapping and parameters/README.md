Activity: Request mapping and parameters
Imagine you’re building a product management system for a small online store. The system needs to provide customers and store administrators with product details, such as names, prices, and descriptions. To make this happen, you’ll need to set up HTTP endpoints that allow users to view, search, and manage product information through an API.

In this lab, you’ll create these API endpoints. You’ll configure endpoints that handle various types of HTTP requests, using annotations like @RequestMapping, @PathVariable, and @RequestParam to capture path and query parameters.

Goal

Build and test HTTP endpoints in a Java controller that handle path and query parameters, enabling efficient data retrieval and management for a product management API.

In this lab, you will: 

Set up and configure HTTP endpoints in a Spring Boot controller.

Use @RequestMapping, @PathVariable, and @RequestParam to handle path and query parameters.

Create CRUD operations to manage product data in a RESTful API.

Test API endpoints with Postman for accurate request and response handling.

Troubleshoot and debug endpoint functionality within a Spring Boot application.

By the end of the lab, you’ll be able to set up endpoints that respond to specific URL patterns and parameters, and you’ll test them using Postman to ensure each endpoint processes data requests and responses accurately.
